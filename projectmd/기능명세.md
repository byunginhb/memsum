## 1. 사용자 흐름 (User Flow)

### 1.1 첫 사용 (Onboarding)
```
App Store → 다운로드 → 첫 실행
   ↓
[Screen 01] 환영 화면 — "찍기만 하세요. Memsum이 알아서."
   ↓
[Screen 02] 가치 시연 (3 컷) — 스크린샷 → 푸시 → 5줄 리포트
   ↓
[Screen 03] Apple/Google 로그인 (필수)
   ↓
[Screen 04] 알림 권한 요청 — "주간 5줄 리포트를 위해 필요해요"
   ↓
[Screen 05] 사진 권한 요청 — "스크린샷 자동 감지를 위해" + 거부 시 수동 가이드
   ↓
[Screen 06] 캘린더 옵션 — Google 연결 / 나중에
   ↓
[Screen 07] 첫 사용 가이드 — "지금 스크린샷 한 번 찍어보세요" (시연용)
   ↓
[Screen 08] 첫 캡처 성공 → 메인 대시보드 진입
```

### 1.2 평일 사용 (Daily Flow)
```
사용자가 스크린샷 찍음
   ↓
[Native] 푸시 알림 — "Memsum에 저장할까요?"
   ↓
사용자 [예] 탭
   ↓
[Native → Server] OCR + 분류
   ↓
[Screen 09] 캡처 확인 다이얼로그
   - 이미지 미리보기
   - OCR 텍스트
   - 자동 분류 결과
   - 이벤트 감지 시 → "📅 캘린더 추가?" CTA
   ↓
사용자 [📅 캘린더 추가] 탭
   ↓
[Server] Google Calendar API 호출
   ↓
[Screen 10] 성공 토스트 — "캘린더에 추가됐어요"
```

### 1.3 일요일 저녁 (Weekly Report)
```
일요일 21:00 KST
   ↓
[Server] 주간 5줄 리포트 생성 + APNs 발송
   ↓
[Native] 푸시 알림 — "📦 이번 주 5줄"
   ↓
사용자 탭
   ↓
[WebView] 5줄 리포트 화면
   - 5개 카드 (제목 + 핵심 + 원본 링크)
   - [👍][👎] 피드백
   ↓
사용자 카드 탭 → 원본 캡처 + 관련 자료
```

### 1.4 검색·재발견 (Discovery Flow)
```
앱 진입
   ↓
[WebView] 메인 대시보드
   - 검색바
   - 최근 캡처 6개
   - 주제별 묶음
   - 캘린더 뷰 (자동 등록 일정)
   ↓
검색바에 "코카콜라" 입력
   ↓
[Server] 벡터 검색 + 키워드 검색
   ↓
[WebView] 검색 결과
   - 매칭 캡처 카드
   - 자동 그룹된 비슷한 자료
   - Pro+ 사용자는 "Ask Your Brain" CTA
```

### 1.5 무료 → 유료 전환 (Upsell Flow)
```
Free 사용자 용량 80% (400MB / 500MB)
   ↓
[Native Push] "이번 주에만 380MB 사용. Lite로 무제한"
   ↓
사용자 탭
   ↓
[WebView] 가격 페이지
   - Free / Lite ₩4,900 / Pro ₩9,900 / Pro Max ₩19,900
   - 사전 결제자: "평생 50% 할인 적용 중"
   ↓
[Lite] 탭
   ↓
[Native] StoreKit 2 결제 시트
   ↓
결제 완료 → 영수증 검증
   ↓
[WebView] "Welcome to Lite" 화면
```

## 3. 데이터 모델 (PostgreSQL — Supabase)

### 3.1 핵심 테이블

#### `users` (Supabase Auth + 확장)
```sql
-- Supabase 기본 auth.users 사용 + public.user_profiles 확장
create table user_profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  email text unique not null,
  name text,
  avatar_url text,
  locale text default 'ko-KR',
  timezone text default 'Asia/Seoul',
  
  -- 구독 정보
  plan text default 'free' check (plan in ('free', 'lite', 'pro', 'pro_max')),
  plan_started_at timestamp with time zone,
  plan_expires_at timestamp with time zone,
  storage_used_bytes bigint default 0,
  storage_limit_bytes bigint default 524288000, -- 500MB
  
  -- 사전 결제자 표시
  is_early_supporter boolean default false,
  early_supporter_code text,
  
  -- 메타
  created_at timestamp with time zone default now(),
  updated_at timestamp with time zone default now()
);

alter table user_profiles enable row level security;

create policy "Users can view own profile"
  on user_profiles for select
  using (auth.uid() = id);

create policy "Users can update own profile"
  on user_profiles for update
  using (auth.uid() = id);
```

#### `captures` — 캡처된 자료
```sql
create table captures (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references user_profiles(id) on delete cascade,
  
  -- 원본 데이터
  source_type text not null check (source_type in ('screenshot', 'share_extension', 'voice', 'manual', 'photo')),
  image_url text, -- Supabase Storage 경로
  image_size_bytes bigint,
  ocr_text text,
  ocr_lang text default 'ko',
  ocr_confidence float,
  
  -- 추출된 의미
  title text, -- AI 생성 제목 (한 줄)
  summary text, -- AI 생성 요약 (한 줄)
  embedding vector(1536), -- pgvector for semantic search
  
  -- 이벤트 감지
  event_detected boolean default false,
  event_title text,
  event_starts_at timestamp with time zone,
  event_ends_at timestamp with time zone,
  event_location text,
  calendar_event_id text, -- Google Calendar event ID
  calendar_added_at timestamp with time zone,
  
  -- 분류
  cluster_id uuid references clusters(id),
  cluster_assigned_at timestamp with time zone,
  
  -- 사용자 액션
  user_starred boolean default false,
  user_archived boolean default false,
  
  -- 메타
  created_at timestamp with time zone default now(),
  captured_at timestamp with time zone default now() -- 디바이스 시간
);

create index idx_captures_user_created on captures(user_id, created_at desc);
create index idx_captures_embedding on captures using ivfflat (embedding vector_cosine_ops);
create index idx_captures_cluster on captures(cluster_id);

alter table captures enable row level security;

create policy "Users can manage own captures"
  on captures for all
  using (auth.uid() = user_id);
```

#### `clusters` — 자동 주제 묶음
```sql
create table clusters (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references user_profiles(id) on delete cascade,
  name text not null, -- AI 자동 명명 ("마케팅 트렌드")
  description text,
  centroid_embedding vector(1536),
  capture_count int default 0,
  
  created_at timestamp with time zone default now(),
  updated_at timestamp with time zone default now()
);

alter table clusters enable row level security;
create policy "Users can view own clusters" on clusters for select using (auth.uid() = user_id);
```

#### `weekly_reports` — 주간 5줄 리포트
```sql
create table weekly_reports (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references user_profiles(id) on delete cascade,
  week_start date not null, -- 월요일
  week_end date not null, -- 일요일
  total_captures int,
  selected_capture_ids uuid[], -- 5개 선정
  
  -- 상태
  sent_at timestamp with time zone,
  opened_at timestamp with time zone,
  
  created_at timestamp with time zone default now(),
  unique(user_id, week_start)
);

alter table weekly_reports enable row level security;
create policy "Users can view own reports" on weekly_reports for select using (auth.uid() = user_id);
```

#### `report_feedback` — [👍][👎] 학습 데이터
```sql
create table report_feedback (
  id uuid primary key default gen_random_uuid(),

  weekly_report_id uuid not null references weekly_reports(id) on delete cascade,
  rating text not null check (rating in ('up', 'down')),

create policy "Users can manage own feedback" on report_feedback for all using (auth.uid() = user_id);
```

#### `oauth_tokens` — 외부 서비스 토큰 (Vault 암호화)
```sql
create table oauth_tokens (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references user_profiles(id) on delete cascade,
  provider text not null check (provider in ('google_calendar')),
  access_token text not null, -- Vault 암호화
  refresh_token text,
  expires_at timestamp with time zone,
  scopes text[],
  
  created_at timestamp with time zone default now(),
  updated_at timestamp with time zone default now(),
  unique(user_id, provider)
);

alter table oauth_tokens enable row level security;
create policy "Users can manage own tokens" on oauth_tokens for all using (auth.uid() = user_id);
```

#### `subscription_events` — 구독 이력 (Apple 영수증)
```sql
create table subscription_events (
  id uuid primary key default gen_random_uuid(),
  user_id uuid not null references user_profiles(id) on delete cascade,
  event_type text not null check (event_type in ('subscribed', 'renewed', 'canceled', 'expired', 'refunded', 'plan_changed')),
  from_plan text,
  to_plan text,
  apple_transaction_id text,
  amount_krw int,
  occurred_at timestamp with time zone not null,
  
  created_at timestamp with time zone default now()
);

create index idx_subscription_user on subscription_events(user_id, occurred_at desc);
alter table subscription_events enable row level security;
create policy "Users can view own subscription" on subscription_events for select using (auth.uid() = user_id);
```

### 3.3 pg_cron 스케줄

```sql
-- 매주 일요일 21:00 KST = 12:00 UTC
select cron.schedule(
  'weekly-report',
  '0 12 * * 0',
  $$
    select net.http_post(
      url := 'https://memsum.supabase.co/functions/v1/generate-weekly-reports',
      headers := '{"Authorization": "Bearer ..."}'::jsonb
    );
  $$
);

-- 매일 새벽 3 시 KST = 18:00 UTC (전일 임시 캐시 정리)
select cron.schedule(
  'daily-cleanup',
  '0 18 * * *',
  $$
    delete from captures where user_archived = true and created_at < now() - interval '30 days';
  $$
);
```

## 4. API 명세

### 4.1 인증
- 모든 API는 `Authorization: Bearer <JWT>` 헤더 필수
- JWT는 Supabase Auth에서 발급 (Apple/Google 로그인)
- RLS가 user_id 자동 격리

### 4.2 캡처 API

#### `POST /api/captures` — 새 캡처 업로드
**Request**
```json
{
  "source_type": "screenshot",
  "image_base64": "...",  // 또는 image_url
  "ocr_text_local": "...", // Vision Framework 1차 OCR
  "captured_at": "2026-05-31T08:30:00+09:00",
  "device_metadata": {
    "model": "iPhone 15 Pro",
    "os": "iOS 17.4"
  }
}
```
**Response**
```json
{
  "capture_id": "uuid",
  "title": "코카콜라 글로벌 캠페인",
  "summary": "지역화 + ESG 결합 새 공식",
  "ocr_text": "...",  // GPT 후처리 결과
  "cluster": {
    "id": "uuid",
    "name": "마케팅 트렌드"
  },
  "event_detected": true,
  "event": {
    "title": "디자인 위크 2026",
```

### 5.1 PhotoLibraryWatcher (iOS)
**역할**: 사진 라이브러리 변경 감지 → JS로 이벤트 발송

**Swift 핵심 코드 구조**
```swift
import Photos
import ExpoModulesCore

public class PhotoLibraryWatcherModule: Module {
  public func definition() -> ModuleDefinition {
    Name("PhotoLibraryWatcher")
    
    Events("onNewScreenshot")
    
    AsyncFunction("startWatching") { () -> Void in
      PHPhotoLibrary.shared().register(self)
    }
    
    AsyncFunction("stopWatching") { () -> Void in
      PHPhotoLibrary.shared().unregisterChangeObserver(self)
    }
  }
}

extension PhotoLibraryWatcherModule: PHPhotoLibraryChangeObserver {
  public func photoLibraryDidChange(_ changeInstance: PHChange) {
    // 새 스크린샷 감지 → sendEvent("onNewScreenshot", payload)
  }
}
```

**React Native 호출**
```typescript
import { PhotoLibraryWatcher } from '@/modules/photo-library-watcher';

useEffect(() => {

}, []);
```

### 5.5 VisionOCR (iOS)
**역할**: 온디바이스 한국어·영어 OCR (1차)

```swift
import Vision

func recognizeText(in image: UIImage, completion: @escaping (String?) -> Void) {
  guard let cgImage = image.cgImage else { return }
  
  let request = VNRecognizeTextRequest { (request, error) in
    let observations = request.results as? [VNRecognizedTextObservation] ?? []
    let text = observations
      .compactMap { $0.topCandidates(1).first?.string }
      .joined(separator: "\n")
    completion(text)
  }
  request.recognitionLanguages = ["ko-KR", "en-US"]
  request.recognitionLevel = .accurate
  
  let handler = VNImageRequestHandler(cgImage: cgImage)
  try? handler.perform([request])
}
```

## 7. 에러 처리 & 재시도

### 7.1 클라이언트 에러 핸들링

| 에러 | 처리 |
|------|------|
| 네트워크 끊김 | 캡처 → 로컬 큐에 저장, 재연결 시 자동 전송 |
| OCR 실패 | 1차 결과만 표시, 사용자가 수동 편집 가능 |
| 캘린더 등록 실패 | 토스트 "잠시 후 다시 시도해주세요" + 재시도 버튼 |
| 권한 거부 | 가이드 화면 + 설정 앱 딥링크 |
| 결제 실패 | Apple 에러 코드 → 친절한 한글 메시지 매핑 |

### 7.2 백엔드 재시도 로직

| 작업 | 재시도 전략 |
|------|------------|
| OpenAI API 호출 | 지수 백오프 (1s, 2s, 4s, 8s, 16s) — 최대 5회 |
| Google Calendar API | 지수 백오프 + Refresh Token 재발급 |
| APNs 발송 | 지수 백오프, 24시간 후 포기 + 인앱 표시 |
| 영수증 검증 | 즉시 1회 + 1분 후 1회 + Dead Letter Queue |

### 7.3 Dead Letter Queue
- 실패한 작업은 `failed_jobs` 테이블에 저장
- 매일 03:00 KST 관리자 알림 (Slack)
- 수동 또는 자동 재처리 흐름

## 8. 분석 이벤트 (PostHog)

### 핵심 추적 이벤트

| 이벤트 | 속성 |
|--------|------|
| `app_opened` | 첫 실행 여부, 콜드/웜 스타트 |
| `onboarding_completed` | 권한 허용 비율 |
| `capture_created` | source_type, OCR 정확도, 응답 시간 |
| `calendar_event_added` | 자동/수동, 정확도 피드백 |
| `weekly_report_opened` | 받은 후 시간차 |
| `feedback_given` | up/down, capture_id |
| `subscription_started` | plan, 무료 체험 여부 |
| `subscription_canceled` | plan, 사용 기간 |
| `storage_limit_reached` | 80%, 100% 도달 |

### 깔때기 (Funnel)
1. 다운로드 → 첫 실행
2. 첫 실행 → 권한 허용
3. 권한 허용 → 첫 캡처
4. 첫 캡처 → 첫 5줄 리포트
5. 첫 5줄 리포트 → Lite 결제

각 단계 전환율 목표: D1 다운로드 → 첫 캡처 60%+
