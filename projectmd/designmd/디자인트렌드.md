# 🎨 Memsum 디자인 트렌드 (2026)

> **결론 한 줄**: Memsum은 **"Calm Glass — 정리 없는 미니멀리즘에 iOS 26 Liquid Glass 외과적 사용 + 한국 토스 결 친근함"** 으로 간다.
>
> 시장은 화려한 고채도·질감·공간 UI로 가지만, Memsum의 페르소나(이수현·노션 다섯 번 접은 사람)는 그 반대쪽에 있다. **트렌드를 따라가지 않는 게 트렌드**다.

---

## 1. 핵심 인사이트 3줄

1. **2026 트렌드의 분기**: 한쪽은 Apple Liquid Glass·고채도·질감 같은 **화려함**, 다른 쪽은 Mem·mymind·Bear 같은 **극한 미니멀**. Memsum은 후자 진영.
2. **Bottom Sheet가 표준 컨테이너로 굳음** — 캡처 확인·설정·필터 모두 Sheet로. Memsum의 캡처 다이얼로그가 시그니처가 됨.
3. **AI는 "기능"이 아닌 "스며듦"** — 한국 트렌드 키워드. Memsum도 "AI로 정리"가 아니라 **"안 보이는 자동화"** 가 페르소나에 더 와닿는다.

### Apple iOS 26 Liquid Glass — Memsum이 받아야 하는가?
**부분 채택, 단 보수적으로.**

Liquid Glass는 iOS 27부터 강제 적용 예정 ([AppleInsider](https://appleinsider.com/articles/26/03/26/stop-holding-out-hope-liquid-glass-will-be-mandatory-in-ios-27)). 그러나 Memsum은:
- 페르소나가 "차분함"을 원하는 사람 → 강렬한 반사·블러는 시각 피로
- 한국 시장 우선 → 한국 사용자는 토스·당근의 **납작한 깨끗함**에 익숙
- WebView와 일관성 → Next.js+shadcn은 Liquid Glass 미지원

**Memsum의 Liquid Glass 적용 규칙**
1. **primary content는 솔리드 라벤더 + 아이보리** (기본 화면)
2. **overlay만 Liquid Glass 결** — 알림 카드, Sheet 상단 핸들 영역, 캘린더 확인 모달
3. **반투명 강도 약하게** — `backdrop-filter: blur(20px)`, `opacity: 0.85` 정도. iOS 26 기본 강도의 60%
4. **iOS 26 미만은 솔리드 폴백** — 호환성 확보

## 4. 노트·캡처 앱 트렌드 (Mem · mymind · Supernotes)

### 공통 패턴
1. **No folders, no tags** — AI 자동 분류 (Memsum과 일치 ✅)
2. **Clutter-free 시각** — 시각적 미니멀 (Memsum 일치 ✅)
3. **Auto-tag while writing** — 작성 중 자동 태그 (Memsum은 캡처 중 자동 분류)
4. **Surface related notes** — 작성 시 관련 노트 자동 추천 (Memsum은 "비슷한 자료" 카드)
5. **No collaboration** — Mem·mymind 모두 협업 약점 (Memsum도 V1 단독 사용에 집중)

### Memsum이 Mem·mymind보다 잘하는 것
- **한국어 OCR + 캘린더 자동 등록** — 글로벌 앱 약점
- **주간 5줄 리포트** — Mem이 "Daily Summary" 시도했으나 약함
- **iOS 26 Liquid Glass 부분 채택** — Mem·mymind는 보수적

### Memsum이 의도적으로 안 따라가는 것
- **Mem의 검색 중심 UI** → Memsum은 "리포트가 알아서 찾아줌"
- **mymind의 비주얼 보드** → Memsum은 리스트·카드 중심 (스크린샷 헤비유저 패턴)

## 5. Memsum 디자인 컨셉 — "Calm Glass"

### 한 줄 정의
> **"한국 토스의 친근한 단순함 위에 iOS 26 Liquid Glass를 외과적으로 얹은 차분한 미니멀리즘."**

### 5가지 디자인 원칙

#### 원칙 1. **솔리드 위 글래스 (Solid Beneath, Glass Above)**
- 기본 화면(홈·자료실·캘린더) = **100% 솔리드 라벤더+아이보리**
- 오버레이(알림·Sheet·캘린더 확인) = **연한 Liquid Glass**
- 카드·리스트 = **솔리드 화이트 + subtle border**
- 의도: 시각 피로 최소화, 페르소나의 "차분함" 욕구 충족

#### 원칙 2. **AI 안 보이게 (Invisible AI)**
- "AI로 정리됨" 같은 라벨 ❌
- 결과만 보여줌 — "지난 주 자료 정리" 같이 사람 친구가 했을 법한 말투
- 자동 클러스터 이름도 사람이 지은 것처럼 ("마케팅 자료" O / "Cluster #3" X)

#### 원칙 3. **하루 한 번 화려한 순간 (One Hero Moment a Day)**
- 평소엔 정적·차분
- **일요일 저녁 9시 5줄 리포트**가 한 주의 시각적 하이라이트
  - 카드 reveal stagger 모션 (디자인시스템.md §7)
  - 코랄 액센트 = "이번 주 베스트"
  - 햅틱 light 1회 (iOS) / vibrate 8ms (Android)
- 이게 일요일을 의식 행위(ritual)로 만든다

#### 원칙 4. **한국 사용자 톤 (Korean Friendly Tone)**
- 모든 카피 친구 말투 — "수현 님" 같은 호칭 가능
- 큰 보라 버튼 1개 + 작은 ghost 버튼 1개 패턴 (토스 결)
- 빈 상태도 따뜻하게 — "아직 캡처가 없어요. 천천히 시작해도 괜찮아요"

#### 원칙 5. **휴먼 인 더 루프 강제 (Always Confirm)**
- 모든 자동 캡처는 1탭으로 확인 받음
- 캘린더 자동 등록도 사용자가 [추가] 누름
- 5줄 리포트도 [👍][👎] 피드백
- 이게 "AI가 마음대로 한다" 불안감 차단

## 6. 화면별 트렌드 적용 가이드

### 6.1 캡처 확인 다이얼로그 (시그니처 화면)
**적용 트렌드**: Bottom Sheet + Liquid Glass (overlay) + Functional Animation

```
┌────────────────────────────────┐
│  [상단: Liquid Glass 핸들 영역]  │ ← 반투명 라벤더 #7C6FE8 + blur 20px
├────────────────────────────────┤
│  [솔리드 영역]                   │
│  📷 캡처 미리보기                │
│  "이 캡처 저장할까요?"           │
│                                  │
│  📅 6/15 (토) 14:00 코엑스       │ ← OCR 결과
│       [Badge: 이벤트 감지됨]     │ ← 코랄 액센트
│                                  │
│  [ 닫기 ]   [ 캘린더에 추가 ]    │ ← Ghost + Primary 1쌍
└────────────────────────────────┘
```

- snapPoints: `['60%']`
- 진입 spring: `damping: 18, stiffness: 220` (gentle)
- 백드롭: `rgba(0,0,0,0.4)` + tap to dismiss

### 6.2 메인 홈 (대시보드)
**적용 트렌드**: Layout Personalization + Bottom Bar Integration

- FAB ❌ — 하단 탭바의 가운데 "+" 캡처 액션
- 사용 패턴 학습: 사진을 더 많이 캡처하면 "자료실" 우선, 약속을 더 많이 잡으면 "캘린더" 우선
- 상단: "오늘 X장 캡처 / 이번 주 N개"
- 솔리드 라벤더 미사용. 베이스는 아이보리 #FFF8F0. 강조만 보라

### 6.3 일요일 5줄 리포트 (Hero Moment)
**적용 트렌드**: Functional Animation + Stagger Reveal + Coral Highlight

- 푸시 알림 진입 시 5장 카드가 80ms 간격 stagger reveal
- 카드 5번 중 가장 "재발견" 된 1장 → 코랄 좌측 보더 4px (Card highlight variant)
- 햅틱 light 1회
- 하단 [👍][👎] 피드백 = "사람의 손이 필요한 순간"

### 6.4 자동 정리 모션 (앱 첫 진입 시 1회)
**적용 트렌드**: Logo Motif Integration + Reanimated

- 9개 점이 무작위 위치에서 3×3 그리드로 spring 이동
- 마지막 우하단 점이 화이트 → 코랄 (200ms)
- 의미: "당신이 안 정리해도 이렇게 정리됩니다"
- 처음 1회만 (2회차부터 짧은 fade)

### 6.5 검색 (자료실)
**적용 트렌드**: Underline Input + Quick Filters + Cluster Surface

- iOS 26 Search field는 Sheet 안에 임베드 (top sticky)
- 입력 시 자동 클러스터 추천 (Mem·mymind 결)
- "비슷한 주제" 측면 카드 (Surface Related Notes)

### 6.6 설정 (List Item 풍부)
**적용 트렌드**: Material 3 / iOS 26 결의 ListItem

- 권한 ON/OFF 스위치 (Memsum 핵심: 사진 자동 감지·캘린더 등록·일요일 푸시)
- 다크모드 system/light/dark 3택
- 사용자 톤 설정 (정중 / 친근 — 페르소나 옵션)

### 6.7 빈 상태 (Empty State)
**적용 트렌드**: Friendly Illustration + Soft Copy

- 로고 9점 그리드 일러스트
- "아직 캡처가 없어요. 천천히 시작해도 괜찮아요."
- 예시 캡처 보기 CTA

## 7. 컬러 미세 조정 (트렌드 반영)

### 기존 팔레트 유지 + 1가지 추가

기존:
- Primary: `#7C6FE8` 라벤더 보라
- Base: `#FFF8F0` 따뜻한 아이보리
- Accent: `#FFB84D` 코랄 옐로우

**추가 (Liquid Glass overlay 전용)**:
- **Glass Tint**: `rgba(124, 111, 232, 0.12)` — 라벤더 12% 알파
  - `backdrop-filter: blur(20px) saturate(140%)`
  - Sheet 상단, 알림 카드 전용

**Memsum의 컬러 절제 룰** (한국 토스·카뱅 결):
- 한 화면에 강조 색은 최대 2개 (보라 + 코랄 중 1개)
- 시스템 색(success/danger 등)은 필요 시만
- "예쁘게 보이려고" 색 추가 금지

## 8. 타이포 미세 조정 (트렌드 반영)

기존 디자인시스템.md의 Pretendard + Inter 유지. 트렌드 반영 추가:

### Hero 사이즈 (5줄 리포트 카드 제목용)
```ts
hero: { size: 34, line: 42, weight: '800' }
// → '700' 로 한 단계 낮춤 (트렌드: 과한 굵기 회피)
```

### Numerical 강조
```ts
display: { size: 28, line: 36, weight: '700' }
// 1,847 같은 큰 숫자 표시용
// font-feature-settings: 'tnum' (등폭 숫자) 적용
```

### 한국어 자모 균형
- Pretendard Variable의 `axes` 활용 — wght 100~900 다이내믹
- 본문: 400, 강조: 500, 제목: 600~700 (800 회피)

## 9. 모션 트렌드 반영 (Functional Helpful)

Muzli 2026: **"마이크로 인터랙션은 시각적 흥미가 아닌 functional helpful로"**

Memsum 적용:
- ❌ 의미 없는 회전·바운스 ❌
- ✅ 상태 전환을 명확히 — 캡처 중·완료·실패 각각 다른 모션
- ✅ 자동 정리 모션 = 시각 외에도 "AI가 일하고 있다"는 신호
- ✅ 5줄 리포트 stagger reveal = "한 주를 천천히 돌아본다" 페이싱

### 신규 모션 토큰 추가

```ts
motion: {
  duration: { instant: 80, fast: 150, base: 200, slow: 300, lazy: 500,
              ritual: 1200 }, // ← 일요일 리포트 stagger 전용
  // ...
}
```

## 11. Anti-trend 선언 (의도적 회피)

Memsum이 **따라가지 않는** 2026 트렌드:

| 트렌드 | 회피 이유 |

| 고채도 강렬한 컬러 | 페르소나 "차분함" 욕구와 충돌 |
| 텍스처·질감 (액체 유리, 동물 털) | 미니멀 컨셉 파괴 |

| 게이미피케이션 | 이수현 페르소나에 부적합 |
| 강제 알림·푸시 인터럽트 | "정리하라" 잔소리 회피 |
| AI 기능 자랑 (라벨·배지) | 한국 트렌드 "AI 스며듦"과 정반대 |

**"트렌드를 따라가지 않는 것이 트렌드인 시대"** — 이수현 페르소나에게는 차분함이 최고의 차별점.

## 12. Claude Code 적용 체크리스트

새 화면 만들 때 Claude Code가 이 문서를 참조하여 다음을 확인:

- [ ] primary content는 솔리드인가? (Liquid Glass는 overlay에만)
- [ ] FAB 사용 안 하고 Bottom Bar에 통합했는가?
- [ ] Bottom Sheet 사용했는가? (모달 대신)
- [ ] 강조 색은 한 화면에 최대 2개인가?
- [ ] 다크모드 자동 전환되는가?
- [ ] 마이크로 인터랙션이 functional한가? (장식 아닌)
- [ ] 한국 토스 결 친근한 카피인가?
- [ ] "AI가 정리했어요" 같은 자랑 라벨이 없는가?
- [ ] 모든 자동 동작에 사용자 [예]/[아니오] 확인이 있는가?
- [ ] 빈 상태 UI가 따뜻한가?
