# Memsum 개발환경 셋업 가이드 (macOS)

> macOS에서 0부터 시작해 **iOS 시뮬레이터 + Android 에뮬레이터에 빈 Expo 앱이 뜰 때까지**의 완전한 셋업 가이드.
> Apple Developer Program($99/년) 결제는 시뮬레이터 단계가 끝난 뒤 진행한다.

---

## 1. 사전 확인

- **macOS 버전**: Sonoma 14 이상 권장 (`애플 메뉴 → 이 Mac에 관하여`)
- **아키텍처**: Apple Silicon (M1/M2/M3) 권장. Intel도 가능
  - 확인: `uname -m` → `arm64`면 Apple Silicon
- **디스크 여유**: **50GB 이상** (Xcode 7GB + Android Studio 10GB + AVD + 캐시)
- **RAM**: 16GB 이상 권장 (시뮬레이터 + 에뮬레이터 동시 실행 시 8GB는 빡빡)

## 2. 필수 설치 (순서대로)

### 2.1 Homebrew
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```
설치 후 `eval "$(/opt/homebrew/bin/brew shellenv)"`를 `~/.zshrc`에 추가.

### 2.2 Node.js 20 LTS (fnm 권장)
```bash
brew install fnm
echo 'eval "$(fnm env --use-on-cd)"' >> ~/.zshrc
source ~/.zshrc
fnm install 20
fnm default 20
```

### 2.3 패키지 매니저 & Git
```bash
brew install git pnpm
```

### 2.4 Watchman
```bash
brew install watchman
```

### 2.5 Xcode 16+
- **App Store**에서 "Xcode" 검색 후 설치 (약 7GB, 30분~1시간)
- 설치 후 한 번 실행해 추가 컴포넌트 다운로드 동의
- 라이선스 동의:
  ```bash
  sudo xcodebuild -license accept
  ```

### 2.6 Xcode Command Line Tools
```bash
xcode-select --install
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
```

### 2.7 CocoaPods
```bash
sudo gem install cocoapods
```

#### 2.7.1 (선택) Ruby 버전 충돌 시 rbenv
```bash
brew install rbenv
echo 'eval "$(rbenv init - zsh)"' >> ~/.zshrc
source ~/.zshrc
rbenv install 3.2.2
rbenv global 3.2.2
gem install cocoapods
```

### 2.8 iOS Simulator
Xcode → Settings → Platforms → iOS 최신 시뮬레이터 다운로드.

### 2.9 JDK 17
```bash
brew install --cask zulu@17
```

### 2.10 Android Studio
- developer.android.com/studio에서 받아 설치
- 첫 실행: **Standard 셋업** 선택 → SDK·Platform-Tools·Build-Tools 자동 설치
- `Settings → Languages & Frameworks → Android SDK`:
  - **SDK Platforms**: Android 14 (API 34)
  - **SDK Tools**: Android SDK Command-line Tools (latest), Android Emulator, Platform-Tools

### 2.11 AVD (Android Virtual Device)
- Android Studio → `Device Manager` → `Create Device`
- 기기: **Pixel 6**, 시스템 이미지: **API 34 (Google Play)**
- 이름: `Pixel_6_API_34`

### 2.12 Expo CLI / EAS CLI
```bash
npm i -g eas-cli
```
(`expo` CLI는 글로벌 설치 불필요. `npx expo` 사용)

### 2.13 Supabase CLI
```bash
brew install supabase/tap/supabase
```

### 2.14 Vercel CLI (선택)
```bash
npm i -g vercel
```

### 2.15 Claude Code
```bash
npm i -g @anthropic-ai/claude-code
claude   # 첫 실행 시 브라우저 인증
```

### 2.16 VS Code (선택)
사용자가 코드를 직접 안 본다면 패스 가능. 로그 보기엔 유용.
```bash
brew install --cask visual-studio-code
```

## 3. 환경 변수 설정 (`~/.zshrc`)

```bash
# Android
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/platform-tools
export PATH=$PATH:$ANDROID_HOME/cmdline-tools/latest/bin

# Java
export JAVA_HOME=$(/usr/libexec/java_home -v 17)

# fnm (위에서 추가했다면 생략)
eval "$(fnm env --use-on-cd)"
```

저장 후:
```bash
source ~/.zshrc
```

## 6. 첫 실행 확인 명령

```bash
node -v                  # v20.x.x
pnpm -v                  # 9.x.x
git --version            # 2.x.x
watchman --version       # 2024.xx.xx.xx
xcode-select -p          # /Applications/Xcode.app/Contents/Developer
pod --version            # 1.15.x
java -version            # openjdk 17.x.x
echo $ANDROID_HOME       # /Users/<you>/Library/Android/sdk
adb devices              # List of devices attached
emulator -list-avds      # Pixel_6_API_34
eas --version            # eas-cli/14.x.x
supabase --version       # 1.x.x
claude --version         # 1.x.x
```

**실패 시 해결**
- `command not found` → `source ~/.zshrc`, 새 터미널
- `xcode-select` 경로가 `/Library/Developer/CommandLineTools` → 2.6 다시
- `adb` 없음 → `ANDROID_HOME/platform-tools` PATH 확인
- `emulator -list-avds` 비어있음 → Android Studio Device Manager에서 AVD 생성

## 7. iOS 시뮬레이터 + Android 에뮬레이터 동시 띄우기

### iOS 시뮬레이터
```bash
open -a Simulator
```
혹은 Xcode → Window → Devices and Simulators.

### Android 에뮬레이터
```bash
emulator -avd Pixel_6_API_34
```
백그라운드 실행: `nohup emulator -avd Pixel_6_API_34 &`

### Expo로 둘 다에 빈 앱 띄우기 (검증)
```bash
npx create-expo-app@latest memsum-test --template
cd memsum-test
npx expo run:ios       # 첫 빌드 5~10분
npx expo run:android
```
양쪽에 "Open up App.tsx..." 화면이 뜨면 성공. 확인 후 `memsum-test` 삭제.

## 8. 흔한 함정 5가지

1. **Xcode 라이선스 미동의**
   - 증상: `Agreeing to the Xcode/iOS license requires admin privileges`
   - 해결: `sudo xcodebuild -license accept`

2. **CocoaPods가 시스템 Ruby와 충돌**
   - 증상: `gem install cocoapods` 권한 오류
   - 해결: 2.7.1처럼 `rbenv`로 Ruby 3.2.2 사용

3. **`ANDROID_HOME` 미설정**
   - 증상: `command not found: adb`
   - 해결: 3장 환경 변수 다시 + 새 터미널

4. **Apple Silicon에서 Rosetta 요구 라이브러리**
   - 증상: 특정 npm 패키지가 `arm64` 미지원
   - 해결: `softwareupdate --install-rosetta --agree-to-license`

5. **Watchman 권한/캐시 문제**
   - 증상: Metro가 파일 변경 미감지
   - 해결:
     ```bash
     watchman watch-del-all
     watchman shutdown-server
     ```
