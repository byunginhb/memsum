# Memsum 디자인 시스템 (Design System v1.0)

> "찍기만 하세요. Memsum이 알아서." — 이 문서는 Claude Code가 Memsum iOS·Android·WebView 코드를 작성할 때 따라야 할 단일 진실 공급원(Single Source of Truth)이다.

---

## 0. 문서 사용 규칙 (READ FIRST)

- 모든 UI 코드는 본 문서의 **토큰**과 **컴포넌트 명세**만 참조한다. 색·간격·폰트 사이즈 하드코딩 금지.
- 새 컴포넌트가 필요하면 먼저 본 문서에 명세를 추가한 뒤 구현한다.
- iOS·Android·WebView 3개 영역의 시각 언어를 **단일 라벤더 톤**으로 통일한다.

---

## 1. 라이브러리 선택

### 1.1 비교표 (Memsum 기준)

| 라이브러리 | WebView 일관성 | iOS·Android | 다크모드 | 성능 | 한국어 | 종합 |
|---|---|---|---|---|---|---|
| **Tamagui** | 중 (shadcn 아님) | 우수 | 우수 (themes) | **최상** (compiler) | 양호 | 8/10 |
| **gluestack-ui v2** | **최상** (shadcn 결) | 우수 | 우수 | 양호 | 양호 | **9/10** |
| NativeWind | 상 (Tailwind 동일) | 양호 | 양호 | 양호 | 양호 | 7/10 |
| RN Paper | 하 (Material 고정) | iOS 부자연 | 우수 | 양호 | 양호 | 5/10 |
| Restyle | 중 (토큰만) | 우수 | 수동 | 우수 | 양호 | 6/10 |
| 자체 구축 | 자유 | 자유 | 수동 | 자유 | 자유 | 6/10 (시간 비용) |

### 1.2 최종 채택: **gluestack-ui v2 + NativeWind**

**이유**
1. WebView가 shadcn/ui + Tailwind인데, gluestack v2는 shadcn 시각 언어를 RN으로 포팅한 라이브러리다 → 네이티브·웹 사용자 경험 단절이 없다.
2. NativeWind로 Tailwind 클래스를 양쪽에서 동일하게 쓸 수 있다 → 마크업 학습 비용 일원화.
3. iOS·Android 둘 다 일급 지원, 다크모드 토큰 스왑 내장.
4. 한 명 개발자(혼자 풀스택) 환경에서 컴포넌트를 직접 만들지 않아도 되어 속도 우선.

**보조 채택**
- 애니메이션: `react-native-reanimated` v3
- 제스처: `react-native-gesture-handler`
- 바텀시트: `@gorhom/bottom-sheet`
- 햅틱: `expo-haptics`
- 아이콘: `lucide-react-native`

## 3. 핵심 컴포넌트 10개 명세

### 3.1 Button
- **용도**: 모든 액션 트리거
- **Variants**: `primary` | `secondary` | `ghost` | `destructive` | `accent`
- **Size**: `sm`(36) / `md`(44, iOS 최소 탭) / `lg`(52)
- **State**: default / pressed / disabled / loading
- **iOS**: `Pressable` opacity 0.7 + `Haptics.selectionAsync()`
- **Android**: ripple `android_ripple={{ color: tokens.primaryMuted }}`

```ts
type ButtonProps = {
  variant?: 'primary' | 'secondary' | 'ghost' | 'destructive' | 'accent';
  size?: 'sm' | 'md' | 'lg';
  loading?: boolean;
  disabled?: boolean;
  leftIcon?: React.ReactNode;
  rightIcon?: React.ReactNode;
  onPress: () => void;
  children: React.ReactNode;
};
```

### 3.2 Card
- **Variants**: `flat` | `elevated` | `outlined` | `highlight`(코랄 좌측 보더 4px — 재발견 표시)
- **Padding**: 기본 `lg`(16), `compact` prop 시 `md`(12)
- **Radius**: `xl`(20) 고정

### 3.3 Input (TextField)
- **Variants**: `outline` | `filled` | `underline`(검색용)
- **Size**: `sm` / `md` / `lg`
- **기능**: label, helperText, errorText, prefix/suffix slot, clear 버튼

### 3.4 List Item
- **구성**: `leading`(아이콘/아바타) · `title` · `subtitle` · `trailing`(chevron/스위치/배지)
- **높이**: 56(단일) / 72(2줄) / 88(3줄)
- **iOS**: 디바이더 좌측 16px 들여쓰기 / **Android**: 풀 너비 디바이더

### 3.5 Badge
- **Variants**: `solid` | `subtle` | `dot`
- **Tone**: `neutral` | `primary` | `success` | `warning` | `danger` | `accent`

### 3.6 Avatar
- **Size**: `xs`(24) / `sm`(32) / `md`(40) / `lg`(56) / `xl`(80)
- **Fallback**: 이니셜 + `primaryMuted` 배경
- **Badge slot**: 우하단 (로고의 코랄 점과 시각적 일관)

### 3.7 Sheet (Bottom Sheet)
- **용도**: 스크린샷 감지 직후 "캡처 확인" 다이얼로그 핵심 UI
- **라이브러리**: `@gorhom/bottom-sheet`
- **snapPoints**: `['25%', '60%', '90%']`
- **모션**: 진입 spring `damping: 18, stiffness: 220`

### 3.8 Toast / Snackbar
- **위치**: iOS 상단(statusBar+12) / Android 하단(Material 결)
- **Tone**: `info` | `success` | `warning` | `danger`

### 3.9 Header (Navigation Bar)
- **iOS**: 큰 제목(34pt) → 스크롤 시 컴팩트(17pt)
- **Android**: 고정 56dp + Title 20sp 좌측 정렬

### 3.10 Empty State
- **구성**: 일러스트(로고 9점 그리드 모티프) · 제목 · 설명 · CTA

## 4. iOS · Android 플랫폼 적응

| 영역 | iOS | Android |
|---|---|---|
| 네비게이션 | 좌측 chevron + Pop 제스처 | 시스템 Back + 상단 좌측 화살표 |
| 모달 | Sheet (Half/Full) | BottomSheet + FullScreenDialog |
| 폰트 폴백 | `-apple-system`, SF Pro | `Roboto`, `sans-serif` |
| 햅틱 | Taptic Engine (`Haptics.impactAsync('light')`) | `Vibration.vibrate(10)` |
| 상태바 | light/dark content | `translucent` + `bgBase` |
| 권한 UI | Apple 시스템 다이얼로그 | Material 다이얼로그 — 사전 설명 권장 |
| 탭바 | 하단 SafeArea 포함 49pt | 56dp BottomNavigation |
| 스위치 | iOS 스타일 토글 | Material 트랙 |
| Material You | 미적용 | Android 12+ 동적 색 옵션 |

```ts
// src/design/theme/platform.ts
export const haptic = (level: 'light' | 'medium' | 'heavy') => {
  if (Platform.OS === 'ios') {
    Haptics.impactAsync(
      level === 'light' ? Haptics.ImpactFeedbackStyle.Light :
      level === 'medium' ? Haptics.ImpactFeedbackStyle.Medium :
      Haptics.ImpactFeedbackStyle.Heavy
    );
  } else {
    Vibration.vibrate(level === 'light' ? 8 : level === 'medium' ? 16 : 32);
  }
};
```

## 5. 다크 모드 전략

- **자동 추적**: 시스템 설정 따름이 기본 + 설정 화면에서 `system | light | dark` 3택
- **토큰 스왑**: 의미 토큰만 참조, 팔레트 직접 참조 금지
- **이미지/아이콘**: 단색은 `currentColor`, 일러스트는 light/dark 2종
- **Memsum 캡처 카드 다크모드**: 썸네일 위 `rgba(255,255,255,0.04)` 오버레이로 OLED 번-인 완화
- **WebView 동기화**: `?theme=dark` 쿼리 또는 `postMessage`로 전달

## 7. 모션·애니메이션

```ts
export const motion = {
  duration: { instant: 80, fast: 150, base: 200, slow: 300, lazy: 500 },
  easing: {
    standard: Easing.bezier(0.2, 0, 0, 1),
    emphasized: Easing.bezier(0.3, 0, 0, 1),
    decel: Easing.out(Easing.cubic),
    accel: Easing.in(Easing.cubic),
  },
  spring: {
    snappy: { damping: 20, stiffness: 300 },
    gentle: { damping: 18, stiffness: 180 },
    bouncy: { damping: 12, stiffness: 220 },
  },
};
```

### Memsum 핵심 모션 4가지
1. **스크린샷 감지 알림**: `translateY: -80 → 0`, opacity 0 → 1, duration 300, easing decel
2. **캡처 확인 시트**: `@gorhom/bottom-sheet` 기본 spring을 `gentle`로 오버라이드
3. **일요일 5줄 리포트 카드**: 스태거 reveal — 80ms 간격, 각 카드 duration 250ms
4. **자동 정리 모션 (로고 9점 정렬)**: 9개 점 무작위 → 3×3 그리드로 spring 이동, delay `i * 50ms`, 마지막 우하단 점 코랄로 색 전환

## 9. Claude Code 작업 룰

### "Button 만들어줘" 트리거 시 체크리스트
1. `src/design/components/Button` 에 이미 있는지 확인
2. 없으면 §3.1 명세 그대로 구현 (variants 5종, size 3종, loading, disabled, leftIcon/rightIcon)
3. 색은 `useTheme()`에서 의미 토큰만 가져옴 (`#7C6FE8` 같은 hex 하드코딩 금지)
4. 다크모드 자동 동작 확인 (light/dark 둘 다 스크린샷)
5. iOS: `Pressable` + `Haptics.selectionAsync()` / Android: `android_ripple`
6. a11y props 누락 금지 (`accessibilityRole`, `accessibilityLabel`, `accessibilityState`)
7. 텍스트는 i18n 키 사용 (`t('button.capture')`). 인라인 한국어 금지
8. Storybook 스토리 1개 이상 추가
